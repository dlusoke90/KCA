Troubleshooting Methodology

1. Define problem. 2. Gather info. 3. Analyze info. 4. Eliminate possible causes. 5. Propose hypothesis. 6. Test. 7. Solve or escalate. Top-down (App→Physical), Bottom-up (Physical→App), Divide and Conquer.

Define problemGather informationAnalyze & hypothesizeTest solution
🛠️

Key Troubleshooting Commands

ping: test connectivity. traceroute: find path. show interfaces: errors, status. show ip route: routing table. debug ip packet: packet-level. show logging: system logs. show tech-support: full dump.

ping: test L3traceroute: find pathshow interfaces: errorsdebug ip packet: detailed
📊

Network Monitoring

SNMP: poll device OIDs for stats. NetFlow: traffic analysis (src/dst/port/bytes). Syslog: event logging (severity 0-7). SPAN/RSPAN: mirror traffic for analysis. CDP/LLDP: discover neighbors.

SNMP: poll statsNetFlow: traffic analysisSyslog: event logsSPAN: traffic mirror
🔄

Common Issues & Solutions

No connectivity: check cables, interface status, IP config. Slow network: check for errors, duplex mismatch, high utilization. Routing issue: check routing table, default gateway. Security: check ACLs, port security.

Check physical firstInterface statusIP configurationACLs last resort
