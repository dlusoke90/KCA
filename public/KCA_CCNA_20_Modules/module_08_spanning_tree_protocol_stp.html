Spanning Tree Protocol (STP)

📚 4 Core Topics 💻 5 CLI Commands ❓ 5 Quiz Questions
Core Concepts
🔄

Why STP?

Redundant links cause Layer 2 loops → broadcast storms → MAC flapping → network failure. STP (802.1D) prevents loops by blocking redundant paths. Creates loop-free logical topology.

Prevents L2 loopsBlocks redundant paths802.1D standardBroadcast storm prevention
👑

STP Election Process

1. Elect Root Bridge (lowest Bridge ID = priority + MAC). Default priority 32768. 2. All ports on Root Bridge = Designated (forwarding). 3. Non-root: elect Root Port (best path to root). 4. Block redundant paths.

Lowest BID = RootPriority + MAC = BIDRoot Port: best to rootDesignated or Blocked
⏱️

STP Port States

Blocking (20s) → Listening (15s) → Learning (15s) → Forwarding. Total convergence: 50 seconds! RSTP (802.1w) converges in ~1-2 seconds. PortFast skips states for access ports.

Blocking 20sListening 15sLearning 15sForwarding ✓
⚡

RSTP & Modern STP

RSTP (802.1w): rapid convergence (<2s). MSTP (802.1s): multiple spanning trees per VLAN group. Cisco's PVST+: separate STP per VLAN. PortFast: skip STP for access ports (use BPDUGuard!).
