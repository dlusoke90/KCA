What is EtherChannel?

Bundles multiple physical links into one logical link. Increases bandwidth, provides redundancy. Up to 8 active links. Appears as single interface to STP. Avoids STP blocking redundant links.

Up to 8 links bundledSingle logical interfaceSTP sees one linkBandwidth aggregation
🤝

LACP (802.3ad)

Industry standard (IEEE). Modes: Active (initiates negotiation), Passive (responds only). Active-Active or Active-Passive works. Passive-Passive = NO channel. Preferred over PAgP.

Active: initiatesPassive: respondsActive+Active ✓Active+Passive ✓
🏢

PAgP (Cisco Proprietary)

Cisco only. Modes: Desirable (initiates), Auto (responds). Desirable-Desirable or Desirable-Auto works. Auto-Auto = NO channel. Being replaced by LACP in modern networks.

Desirable: initiatesAuto: respondsDesirable+Desirable ✓Auto+Auto = NO
⚙️

EtherChannel Configuration

interface range Gi0/1-4 → channel-group 1 mode active → interface Port-channel1 → configure as normal interface. Load balancing: default src-dst-mac, can change to IP-based.
