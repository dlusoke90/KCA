Core Concepts
📋

Types of Routes

Connected (C): directly attached networks. Static (S): manually configured. Dynamic: learned via OSPF/BGP. Default route (S*): 0.0.0.0/0 — matches all traffic. Local (L): exact IP of interface.

C: ConnectedS: StaticDynamic: OSPF/BGPS*: Default route
⚙️

Static Route Syntax

ip route [network] [mask] [next-hop|exit-interface]. Example: ip route 10.0.0.0 255.0.0.0 192.168.1.2. Floating static: add higher AD (ip route ... 150). Default: ip route 0.0.0.0 0.0.0.0 [next-hop].

ip route network maskNext-hop or interfaceAD default = 1Floating static backup
🔢

Administrative Distance

Trustworthiness metric. Lower = more preferred. Connected=0, Static=1, EIGRP=90, OSPF=110, RIP=120, External BGP=20, eBGP=20. Used when multiple sources for same route.

Connected = 0Static = 1OSPF = 110RIP = 120
🔄

Recursive vs Direct

Recursive: next-hop IP (router must look up exit interface). Direct: exit interface specified. Fully specified: both next-hop + interface (most efficient). Point-to-point: interface only is fine.

Recursive: next-hop IPDirect: exit interfaceFully specified: bothP2P: interface OK
