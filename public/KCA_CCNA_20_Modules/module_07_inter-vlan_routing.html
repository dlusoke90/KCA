Inter-VLAN Routing

📚 4 Core Topics 💻 5 CLI Commands ❓ 5 Quiz Questions
Core Concepts
🔌

Router on a Stick

Single physical link between router and switch configured as trunk. Router creates subinterfaces for each VLAN. Subinterface format: Gi0/0.10 for VLAN 10. Bottleneck at the single link.

Single trunk linkSubinterfacesGi0/0.10 → VLAN10Simple but bottleneck
🔄

Layer 3 Switch (SVI)

Most modern approach. Switch Virtual Interface (SVI) = virtual interface per VLAN. ip routing command enables routing. Much faster than router-on-a-stick. No external router needed.

ip routing commandSVI = virtual interfaceinterface vlan 10Faster than RoaS
⚙️

Subinterface Configuration

interface Gi0/0.10 → encapsulation dot1Q 10 → ip address 192.168.10.1 255.255.255.0. Repeat for each VLAN. Physical interface must be 'no shutdown' with no IP.

encapsulation dot1QIP per subinterfaceno shutdown on physicalNo IP on physical
🏗️

SVI Configuration

interface vlan 10 → ip address 192.168.10.1 /24 → no shutdown. Requires 'ip routing' globally. VLANs must exist and have active ports. Default gateway for hosts = SVI IP.

interface vlan 10ip routing globalHosts use SVI as GWVLAN must be active
