Core Concepts

Why Automation?

Traditional CLI config is manual, error-prone, slow. Automation: consistent configs, faster deployment, version control. Python + Netmiko/NAPALM/Nornir for Cisco. APIs replace CLI in modern networks.

Consistent configsFaster deploymentVersion controlError reduction
📡

REST APIs

HTTP methods: GET (read), POST (create), PUT (update), DELETE (remove). JSON/XML data format. Status codes: 200 OK, 201 Created, 400 Bad Request, 401 Unauthorized, 404 Not Found, 500 Server Error.

GET: readPOST: createPUT: updateDELETE: remove
🐍

Python for Networking

Netmiko: SSH connections to network devices. NAPALM: multi-vendor abstraction. Nornir: parallel execution framework. Requests: REST API calls. Ansible: agentless automation (YAML playbooks).

Netmiko: SSH automationNAPALM: multi-vendorNornir: parallelAnsible: YAML playbooks
🏗️

Network Models

SDN: Software Defined Networking — separate control and data plane. Controller manages all devices. Cisco DNA Center: intent-based networking. OpenFlow: SDN protocol. Overlay/Underlay: VXLAN, ACI.

SDN: separate planesController: centralizedDNA Center: intent-basedVXLAN: overlay network
