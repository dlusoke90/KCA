Core Concepts

Why NAT?

Network Address Translation conserves IPv4 addresses. Private IPs (RFC1918) not routable on internet. NAT translates private→public. Types: Static (1:1), Dynamic (pool), PAT/overload (many:1).

Conserve IPv4Private→PublicStatic: 1:1 mappingPAT: many:1 (most used)
⚙️

Static NAT

Permanent one-to-one mapping. Each private IP gets a dedicated public IP. Used for servers (web, mail) needing consistent public IP. ip nat inside source static 10.0.0.1 203.0.113.1.

1:1 permanent mappingServer accessip nat inside staticAlways same public IP
🔀

Dynamic NAT & PAT

Dynamic: pool of public IPs, first-come-first-served. PAT (overload): single public IP, differentiates by port number. Most home/office NAT = PAT. ip nat inside source list 1 interface gi0/1 overload.

Dynamic: IP poolPAT: port-basedoverload keywordMost common = PAT
🔌

NAT Inside/Outside

Inside local: private IP of host. Inside global: public IP after translation. Outside local: destination as seen from inside. Outside global: actual destination public IP. Mark interfaces: ip nat inside / outside.

Inside local: privateInside global: publicip nat insideip nat outside
