DHCP DORA Process

Discover: client broadcasts. Offer: server offers IP. Request: client requests offered IP. Acknowledge: server confirms. Client gets IP, mask, gateway, DNS. Lease time configurable.

D: Discover (broadcast)O: Offer (server)R: Request (client)A: Acknowledge (server)
⚙️

DHCP Server Configuration

ip dhcp pool LAN → network 192.168.1.0 /24 → default-router 192.168.1.1 → dns-server 8.8.8.8 → lease 7. Exclude static IPs: ip dhcp excluded-address 192.168.1.1 192.168.1.10.

ip dhcp pool NAMEnetwork + maskdefault-routerdns-server
🔄

DHCP Relay Agent

Routers block broadcasts by default. ip helper-address forwards DHCP broadcasts to server. Configured on router interface facing clients. Allows one DHCP server for multiple subnets.

ip helper-addressForwards broadcastsServer on different subnetUnicast to DHCP server
🛡️

DHCP Snooping

Security feature. Distinguishes trusted (uplink) from untrusted (client) ports. Blocks rogue DHCP servers. Builds binding table (IP+MAC+port). Required for Dynamic ARP Inspection (DAI).

Trusted vs untrusted portsBlocks rogue DHCPBinding tableRequired for DAI
