Core Concepts

What is a Network?

A network is a collection of devices connected together to share resources and communicate. Networks range from 2 devices at home to billions on the internet. Types: • LAN (Local Area Network) — same building/campus • WAN (Wide Area Network) — across cities/countries • MAN (Metro Area Network) — city-wide • PAN (Personal Area Network) — Bluetooth, near you • WLAN (Wireless LAN) — WiFi network

LAN: localWAN: wideMAN: metroWLAN: wireless
🖥️

End Devices — Host, Client, Server

End device, Host, Client, and End User all mean the same thing — a device that sends or receives data. • Host: any device with an IP address (PC, phone, printer) • Client: sends requests (your browser asking for a website) • Server: responds to requests (web server returning the page) • The same PC can be BOTH client and server • Examples: PC, laptop, smartphone, IP phone, tablet, printer, IP camera

Host = End DeviceClient: sends requestsServer: respondsCan be both
🔌

Network Devices — Infrastructure

These devices connect end devices and forward traffic: 🔵 Hub (Layer 1): broadcasts to ALL ports — legacy, creates collisions 🟢 Layer 2 Switch: forwards by MAC address, creates VLANs, no routing 🟡 Layer 3 Switch: does both switching AND routing, has SVIs, faster than router for inter-VLAN 🔴 Router: connects different networks, routes by IP address, default gateway 🛡️ Firewall: filters traffic by rules (ACL), stateful inspection, sits at network edge 📶 Wireless Access Point (WAP): connects WiFi devices to wired network 📡 Wireless Modem/Router: home device that combines modem + router + WAP + switch

Hub: L1 broadcastL2 Switch: MAC-basedL3 Switch: routes+switchesRouter: IP routingFirewall: securityWAP: wireless
🔌

Network Cables & Connectors

Copper Cables: • UTP Cat5e: up to 1Gbps, 100m, RJ-45 connector • UTP Cat6: up to 10Gbps (55m), 100m at 1Gbps, RJ-45 • UTP Cat6a: up to 10Gbps at 100m, RJ-45 • Straight-through: PC→Switch, Router→Switch • Crossover: Switch→Switch, PC→PC (legacy, auto-MDI-X fixes this) • Console/Rollover: RJ-45 to DB-9, for Cisco console access Fiber Cables: • Single-mode (SMF): yellow jacket, long distance (up to 40km), LC/SC connector • Multi-mode (MMF): orange/aqua jacket, short distance (550m), LC/SC connector Connectors: RJ-45 (copper), LC (fiber - small), SC (fiber - square), SFP (transceiver module)

UTP Cat5e/6: copperRJ-45 connectorStraight-through most commonLC/SC: fiber
📚

The OSI Model (7 Layers)

Mnemonic: 'Please Do Not Throw Sausage Pizza Away' Layer 7 — Application: HTTP, FTP, DNS, SMTP (user-facing protocols) Layer 6 — Presentation: encryption, compression, format conversion (SSL/TLS) Layer 5 — Session: establish/maintain/terminate sessions (NetBIOS) Layer 4 — Transport: TCP/UDP, port numbers, segmentation Layer 3 — Network: IP addressing, routing (Router, L3 Switch) Layer 2 — Data Link: MAC addresses, frames, switching (L2 Switch) Layer 1 — Physical: bits, cables, signals (Hub, NIC, cable)

L7: ApplicationL4: Transport (TCP/UDP)L3: Network (IP)L2: Data Link (MAC)L1: Physical (bits)
🕸️

Network Topologies

Physical topology = how cables are laid out Logical topology = how data flows • Bus: all devices on one cable (legacy, single point of failure) • Star: all connect to central switch (modern standard) ✓ • Ring: devices in a loop (Token Ring — legacy) • Mesh: every device connects to every other (WAN redundancy) • Full Mesh: n*(n-1)/2 links — maximum redundancy • Partial Mesh: some redundant links — balance of cost/redundancy • Hybrid: combination (most enterprise networks = star + partial mesh)

Star: modern LANMesh: WAN redundancyBus: legacyHybrid: enterprise
